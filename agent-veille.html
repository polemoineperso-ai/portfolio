Problème résolu

Le bruit avant le signal

La recherche d'emploi senior en communication génère chaque jour des dizaines d'offres sur des plateformes disparates. Les trier manuellement est chronophage et peu fiable : même avec des filtres, 90% des offres ne correspondent pas réellement au profil recherché.

L'objectif était d'automatiser intégralement cette veille pour ne recevoir chaque matin que les 2 ou 3 offres vraiment pertinentes, avec une justification IA et des actions en un clic directement depuis l'email.

Architecture

Comment ça fonctionne

⏰
Déclenchement automatique — 7h00
Le planificateur Windows (compte SYSTEM) lance run.bat chaque matin. Même si le PC était en veille, la tâche s'exécute au réveil. Fonctionne sans session utilisateur active.
🔍
Scraping multi-sources en parallèle
Trois connecteurs tournent simultanément : France Travail API sur 8 départements Île-de-France, Adzuna API avec pagination, Cadremploi via flux RSS avec filtrage par mots-clés côté client.
~130 offres France Travail · Adzuna · Cadremploi RSS
🔁
Déduplication intelligente
Trois fichiers JSON gèrent l'état : seen.json (déjà traitées), applied.json (archivées), saved.json (à relire). Une offre sauvegardée revient chaque jour jusqu'à action de l'utilisateur.
🤖
Scoring IA — Claude Sonnet
Chaque offre est soumise à Claude avec un profil détaillé : titre ciblé, zone géographique, compétences clés, secteurs exclus, contrats acceptés. Le modèle retourne un score /10, un tag et une justification en une phrase.
Score · Tag prioritaire / à regarder / ignorer · Justification
📧
Email digest HTML — actions en un clic
Envoi via Gmail API (OAuth 2.0). Chaque carte d'offre contient le score, la justification IA, et trois boutons : archiver, sauvegarder, ouvrir dans le générateur de candidature avec le texte pré-rempli.
Défis techniques

Les vrais problèmes à résoudre

Exécution sous compte SYSTEM Windows
La tâche planifiée tourne sans session utilisateur. Contraintes : pas d'accès à C:\Users\ pour les packages pip (environnement virtuel dans C:\ProgramData\), tokens OAuth stockés séparément, redémarrage propre du serveur local géré dans le batch.
Bug WMIC / encodage UTF-16LE
wmic os get localdatetime sort en UTF-16LE avec espaces dans un contexte SYSTEM, générant des noms de fichiers de log invalides et des échecs silencieux. Solution : date via PowerShell Get-Date -Format yyyyMMdd.
Injection HTML sécurisée dans le générateur
Le générateur de candidature contient </body> dans des chaînes JavaScript. Utilisation de rfind() plutôt que replace() pour cibler uniquement la vraie balise de fermeture.
Fiabilité sur laptop en veille
Deux paramètres critiques : DisallowStartIfOnBatteries=false et StartWhenAvailable=true. La tâche se déclenche sur batterie et se lance dès le réveil si le PC était en veille à 7h00.
